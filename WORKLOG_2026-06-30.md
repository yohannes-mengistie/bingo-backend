## What we shipped today

### 1. Payment verification via verify.leul.et
- New package `internal/payment/verifier.go` — POSTs to `https://verifyapi.leulzenebe.pro/verify`
  with `x-api-key`, normalizes the response into `domain.PaymentVerificationResult`.
- Wired into deposit flow (`internal/usecase/wallet.go`): when a verifier is
  configured, a deposit is verified and **auto-credited instantly**; otherwise it
  stays `pending` for manual admin approval.
- Commit: `adf460a`.

### 2. Fixed nil-interface bug (important)
- `NewVerifier` returns `domain.PaymentVerifier` (interface), not `*Verifier`.
  A typed-nil `*Verifier` boxed into an interface is NOT `== nil`, which would have
  made `uc.paymentVerifier != nil` always true and **broken all deposits when no
  API key was set**. Returning the interface type fixes it.
- Part of commit `adf460a`.

### 3. Fallback to manual approval on verifier outages
- `domain.ErrVerifierUnavailable` sentinel. Infrastructure failures (network,
  timeout, 5xx, 401/403, 429) → deposit left **pending** for manual approval
  (not rejected). Definitive bad-receipt verdicts → still rejected.
- Commit: `f9ce42e`.

### 4. Removed CBE entirely — Telebirr is the only payment method
- Backend: dropped `PaymentMethodCBE`, CBE/suffix verifier logic, `VERIFY_CBE_SUFFIX`
  config + render var. Validation now Telebirr-only.
- Frontend: removed CBE from method selector (now no selector — single fixed
  Telebirr), `PAYMENT_ACCOUNTS`, `PaymentMethod` type, i18n hints, `VITE_CBE_ACCOUNT`.
- **DB CHECK constraints left intact** (`IN ('CBE','Telebirr')`) so historical CBE
  rows stay readable and any pending CBE deposits can still be approved/rejected.
- Commits: backend `b88af2e`, frontend `9ae30a1`.

### 5. Telebirr service-fee fix (settledAmount)
- Telebirr charges the sender a fee, so the receipt's `totalPaidAmount` = amount + fee
  (e.g. 21 for a 20 deposit). The verifier was reading `totalPaidAmount`, forcing
  players to type 21.
- Fix: verifier now prefers **`settledAmount`** (net amount the house receives = 20).
  Wallet is credited the verified settled amount. Match tolerance widened to 1 birr
  so the player just types the amount they sent.
- Commits: `2a15084` (surface amounts in error), `c9b92c3` (settledAmount).

### 6. Account-binding (security — prevents claiming someone else's receipt)
- Verifier now checks the receipt's `creditedPartyAccountNo` matches the configured
  house Telebirr number (`VERIFY_TELEBIRR_ACCOUNT`). Receipts mask middle digits
  (`2519****9691`), so matching is on the visible trailing digits (last 4+).
- Without this, ANY valid Telebirr receipt (money sent to a third party) could be
  claimed. Now rejected: "receipt was paid to a different account…".
- Commit: `75b7e87`.

### 7. House Telebirr number set
- Telebirr deposit number: **`0997709691`** (name "Habesha Bingo").
- Frontend `.env.production` + code default updated; bundle confirmed serving it.
- Commit: frontend `167fc6a`.

## Current verification pipeline (a deposit auto-credits only if ALL pass)
1. Receipt is real & status Completed (verify.leul.et).
2. Paid to the house account — `creditedPartyAccountNo` matches `0997709691`.
3. Amount matches `settledAmount` (fee excluded); wallet credited that net amount.
4. Reference not already used (duplicate check + partial unique index).

- Verifier unreachable → deposit goes **pending** for manual admin approval.
- Definitive bad receipt (wrong account/amount, fake, not found) → rejected (HTTP 400).

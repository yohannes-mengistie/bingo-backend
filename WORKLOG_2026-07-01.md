## What we shipped today

### 1. Automatic BINGO (no manual claim)
- The game no longer waits for a player to tap BINGO. After **each drawn
  number**, `GameUseCase.checkAutoBingo` scans every active card and finalizes
  the game the instant one completes a valid pattern.
- Works because auto-daub means marks == drawn numbers on the card, so a card
  wins exactly when `ValidateBingo(card, drawnSet)` is true.
- Wired into the `drawNumbers` loop (returns/stops drawing once resolved).
- `internal/usecase/game.go`.

### 2. Pot splitting among co-winners
- Every card completing on the **same draw** is a co-winner and shares the prize
  pool evenly. `splitPot(pool, n)` — santim-rounded, remainder folded into the
  first winner so the credited total is **exactly** the pool (never more).
- One user holding several winning cards gets one share **per card**.
- Deterministic tie-break / primary winner: earliest joiner, then lowest card ID.

### 3. Shared winner-resolution path
- Extracted `finalizeWinners(game, winners[])` — the single path used by BOTH
  the auto-check and the (now-vestigial) manual `POST /games/:id/bingo`.
- Its atomic `ClaimWinner` guard (DRAWING→FINISHED) still prevents any double
  payout if a manual claim races the auto-check.

### 4. Per-card winner tracking (schema)
- Migration `017_add_game_player_winnings.sql`: adds
  `game_players.is_winner BOOLEAN` and `prize_won NUMERIC(12,2)` (idempotent
  `ADD COLUMN IF NOT EXISTS`). Also added to `migrations/init.sql` for fresh
  installs.
- `games.winner_id` still holds the single **primary** winner (recent-winners
  feed, backward-compatible queries); the new columns record each winning card
  and its exact share.

### 5. Winner card exposure (live + reconnect)
- Live `WINNER` WebSocket event now carries `winners[]` (each `{user_id,
  winner_name, prize, card_id, marked_numbers}`), plus `prize_pool` and `split`.
  Top-level fields still mirror the primary winner (backward compatible).
- Finished-game **INITIAL_STATE** now includes the winning card(s) via new
  `GameUseCase.GetGameWinners` / `gameRepo.FindWinningCards`, reconstructing marks
  from the drawn set. So a client reconnecting AFTER the transient event can
  still render the post-game winner screen.

### 6. Game-history regression fix (important)
- `FindGamesByUserID` now derives per-user win/amount from the per-card flags,
  **falling back to `winner_id` == user (full pool)** for games finished BEFORE
  this change (is_winner was never backfilled). Without the fallback, every
  historical win would have shown as a loss.

### 7. Frontend (miniapp) — `~/bingo-miniapp`
- `WINNER` + finished `INITIAL_STATE` handlers read the new `winners[]` (with
  fallback to legacy single-winner fields); detect if the viewer is any winner
  and use their own share for "You won".
- `ResultOverlay` **lists all winners** with their share + per-winner "view card"
  (tap to verify each). Design chosen: list-all + tap-to-view.
- Manual BINGO button removed → passive "🎉 BINGO" badge lights the card (server
  auto-declares). Cleaned up orphaned `onClaim`/`claimPositions`/`ApiError`/
  `Button`/`useToast` (noUnusedLocals).
- Types (`GameWinner`, extended `WinnerData`) + i18n (`en`/`am`):
  `winnersLabel`, `potSplit`, `viewCard`.

## How to run the integration test (local)
Local dev Postgres is at `127.0.0.1:5432` (the `.env` `DB_HOST=postgres` is the
docker service name; override to `127.0.0.1`). Redis at `127.0.0.1:6379`, no pw.
```
DB_HOST=127.0.0.1 DB_USER=postgres DB_PASSWORD=<.env> DB_NAME=bingo DB_SSLMODE=disable \
REDIS_HOST=127.0.0.1 REDIS_PORT=6379 \
  go test -tags=integration -run Integration ./internal/usecase/ -v
```
